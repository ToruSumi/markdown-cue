# Quickstart: Additional Markdown Snippets

**Feature**: [spec.md](spec.md) | **Date**: 2026-05-21

## ローカル確認手順

### 前提条件

- Node.js で依存関係がインストール済み
- TypeScript compile が通ること
- Markdown Cue 拡張を Extension Development Host で起動できること

### テスト 1: 下線 snippet を補完候補から挿入する

1. リポジトリで `npm test` を実行する
2. VS Code でこの拡張を起動し、Markdown ファイルを開く
3. `;under` など下線候補に到達できる入力を行う
4. 下線候補を選択する
5. `<u>text</u>` 形式で挿入され、最初の編集位置が `text` にあることを確認する
6. 候補一覧表示後、方向キーと確定キーの 2 操作以内で挿入完了できることを確認する

**期待結果**: 下線 snippet が補完候補から正しく挿入され、候補一覧表示後 2 操作以内で完了できる

### テスト 2: 折りたたみ snippet を補完候補から挿入する

1. Markdown ファイルで `;details` など折りたたみ候補に到達できる入力を行う
2. 折りたたみ候補を選択する
3. `<details>` と `<summary>summary</summary>`、`body`、`</details>` を含む複数行テンプレートが挿入されることを確認する
4. Tab で `summary` と `body` を順に編集できることを確認する

**期待結果**: 折りたたみ template は実改行で挿入され、`\\n` 文字列は表示されない

### テスト 3: コマンド経由でも同じ 2 項目が使える

1. コマンドパレットから `Markdown: Insert Syntax` を実行する
2. 一覧に下線と折りたたみが表示されることを確認する
3. それぞれを選択して、補完経路と同じ snippet が挿入されることを確認する

**期待結果**: Quick Pick 一覧が共有カタログと一致し、2 項目とも挿入できる

### テスト 4: 既存候補へ回帰がないことを確認する

1. `;table` と `;math` を入力して既存候補を表示する
2. 既存候補が引き続き表示されることを確認する
3. `npm test` の snippets/completionProvider 関連テストが通ることを確認する
4. 下線と折りたたみが既存候補一覧の末尾に追加され、既存候補の順序が変わっていないことを確認する

**期待結果**: 新規追加で既存候補の順序や複数行挿入が壊れない
